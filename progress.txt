## Codebase Patterns
- Use `python3` not `python` to run commands (macOS setup)
- Pre-existing mypy error in `src/billfox/extract/docling.py:66` (DocumentStream export) - not a blocker
- Test patterns: use `unittest.mock.patch("pydantic_ai.Agent")` to mock Agent creation, `AsyncMock` for async agent.run()
- Use `FakeRunResult(output=...)` dataclass pattern for mocking pydantic-ai run results
- All test classes in `tests/test_parse_llm.py` follow `TestLLMParser*` naming
- Backup providers follow async pattern: sync method `_backup_sync()` wrapped in `asyncio.to_thread()`
- Date folders use UTC and zero-padded month/day: `YYYY/MM/DD`
- Mock `datetime` in backup modules by patching `billfox.backup.<module>.datetime`
- Ollama models use `pydantic_ai.models.openai.OpenAIModel` + `pydantic_ai.providers.openai.OpenAIProvider` with `{base_url}/v1/`
- For lazy imports in `_get_agent()`, patch at source (`pydantic_ai.models.openai.OpenAIModel`) not at the importing module
- `split(":", 1)` on model string handles tags like `llama3.2:7b` correctly
