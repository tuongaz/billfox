## Codebase Patterns
- Python binary is at `.venv/bin/python` (use full path, `python` is not on PATH)
- Use `uv pip install -e ".[dev]"` to install all deps including dev extras
- CLI lives in `src/billfox/cli/app.py`, entry point is `billfox.cli:app` (Typer app)
- Shared CLI helpers live in `src/billfox/cli/_helpers.py`; import via `import billfox.cli._helpers as _helpers`
- Config dir is `~/.billfox/`, config file is `~/.billfox/config.toml`
- Tests use `pytest` with `asyncio_mode = "auto"`, runner is `CliRunner` from `typer.testing`
- Ruff enforces import sorting (I001) - keep `import` blocks properly separated
- mypy runs in strict mode; use `# type: ignore[...]` sparingly
- No `--timeout` flag for pytest in this project
- dotenv loading happens at module import time in app.py; wrapped in try/except ImportError so it's optional
- Models live in `src/billfox/models/` package; import from `billfox.models.receipt`
- Mock helpers at `billfox.cli._helpers.xxx` (not `billfox.cli.app._xxx`) since `app.py` uses module-level access
- New CLI commands use import-and-register pattern: `from billfox.cli.xxx import xxx as xxx_command; app.command("xxx")(xxx_command)`
- Receipt command always creates SQLiteDocumentStore — mock it in tests even when `--store` is not passed
