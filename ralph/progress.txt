## Codebase Patterns
- Use `python3` not `python` ŌĆö only python3 is available on this system
- Use `from __future__ import annotations` in all source files for forward references
- Protocols use `@runtime_checkable` decorator and live in `_base.py` files within each subpackage
- Covariant TypeVar (`T_co`) required for generic Protocol type parameters (mypy strict)
- Ruff TCH rules disabled ŌĆö runtime imports needed for @runtime_checkable protocols and dataclass fields
- mypy overrides for third-party packages use separate `[[tool.mypy.overrides]]` blocks per module
- Each subpackage has `__init__.py` re-exporting public names via `__all__`
- Use `Image.Resampling.BILINEAR` not `Image.BILINEAR` (deprecated in Pillow>=10)
- Reference code uses cv2 but billfox uses Pillow ŌĆö convert BGRŌåöRGB with `[:, :, ::-1]`
- Wrap sync SDK calls with `asyncio.to_thread()` for async protocol implementations
- Don't use `type: ignore[import-untyped]` for installed dev dependencies ŌĆö mypy flags it as `unused-ignore`
- Lazy-imported client getters (`_get_client`) must return `Any`, not `object` ŌĆö mypy can't see methods on `object`
- Use `datetime.UTC` not `datetime.timezone.utc` ŌĆö ruff UP017 enforces the alias
- FTS5 content sync tables need manual `INSERT ... VALUES('delete', ...)` before row updates/deletes
- `event.listen(engine.sync_engine, "connect", callback)` for SQLite pragma setup with async engines
- FTS5 `bm25()` returns negative scores (lower = better); negate for fusion (higher = better)
- sqlite-vec vec0 `distance` is cosine distance; convert to similarity with `1 - distance`
- Vec0 row IDs use `document_id:field_name` format ŌĆö split on `:` to extract document_id
- Do NOT use `from __future__ import annotations` in CLI modules ŌĆö breaks typer's runtime annotation inspection
- typer requires `>=0.13` for click 8.3+ compatibility (`make_metavar` signature change)
- `python -m billfox` requires `__main__.py` at package root (`src/billfox/__main__.py`), not in cli/
- Patch lazy-imported classes at their source module (e.g. `billfox.source.local.LocalFileSource`) for test mocking
- Config helpers use `_get_config_dir()` function (not constant) to enable test patching with `tmp_path`
- `tomli_w.dump()` takes binary file handle (`"wb"`); `tomllib.load()` also takes binary (`"rb"`)
- Always use `@pytest.mark.asyncio async def` for async tests ŌĆö `asyncio.get_event_loop().run_until_complete()` is fragile across test runs
- Long `with` statement combining uses parenthesized form: `with (\n    ctx1,\n    ctx2,\n):`
